Upgrade the current “Application Logs” module to provide reliable ingest, storage, querying, visualization, and lifecycle management of logs in a single Replit project (usable via shell, file system, console, and package manager). Keep the solution stack‑agnostic and runnable with one command.

Objectives

Improve troubleshooting speed, correctness, and safety using evidence‑based logging/observability practices.

Preserve the existing UI layout (KPI cards, Filters, Logs list) while adding robust back‑end APIs, error grouping, and alerting.

Evidence‑Based Methods (must implement)

Structured logging (JSON) with fixed schema; avoid free‑text only.

Standard levels: trace, debug, info, warn, error, fatal; enforce server‑side filtering.

Context enrichment: ISO8601 UTC timestamp, service/module, category, requestId/correlationId, userId (if present), sessionId, env, version/commit, host.

Redaction: scrub PII/secrets via configurable allow/deny rules; log redaction actions.

Cardinality control: cap high‑cardinality fields, hash long/unique values.

Sampling & rate limiting: per‑level sampling (e.g., trace/debug) and burst throttling with metrics.

Retention & rotation: size/time‑based rotation; archive with TTL; restore path.

Error grouping: fingerprint by normalized stack + message template; track counts/first/last seen.

Health & anomalies: rolling error‑rate alerts using z‑score or EWMA; persistence of alert events.

Validation: schema validation on ingest; quarantine malformed entries.

Exportability: NDJSON/CSV streaming export honoring current filters.

Data Model

LogEntry { id, timestamp, level, message, category, service, requestId, userId, sessionId, env, version, metadata<object>, archived<bool> }

ErrorGroup { id, fingerprint, firstSeen, lastSeen, count, sampleLogId }

Settings { retentionDays, redactionRules[], sampleRates{level}, alertThresholds }

Alert { id, createdAt, scope(service|category), metric, threshold, value, status }

APIs

POST /logs (single/batch): validate → redact → sample/rate‑limit → persist.

GET /logs?level=&category=&archived=&q=&from=&to=&limit=&cursor= (cursor pagination).

GET /stats/overview → totals, active, archived, recent errors (last 30d), top categories.

GET /stats/timeseries?metric=(count|errorRate)&interval=…

POST /groups/rebuild → async regrouping.

PATCH /logs/:id → archive/unarchive.

DELETE /logs/:id

POST /export → NDJSON/CSV stream for current filter set.

POST /logs/test?count=N → generate synthetic logs for demo.
